本文档汇总所有环境、安装、运行与常见问题，便于在不同机器或隔一段时间后按步骤操作，无需再问一遍。

## 2. API Key 配置（AI 生成与运行时叙事用）

任选一种即可（优先级从高到低）：

1. **根目录或 game 下 .env**：  
   `VITE_AIHUBMIX_API_KEY=sk-你的key` 或 `AIHUBMIX_API_KEY=sk-你的key`  
   游戏与流水线会先读 `game/.env`，再读根目录 `.env`。
2. **game/public/config.json**（不要提交）：  
   `{ "aihubmixApiKey": "sk-..." }`
3. **根目录 api_key.txt**：  
   文件内任意一行包含 `sk-...` 即可被解析（脚本与 Electron 会读）。

## 3. 游戏完整流程

### 3.1 环境

- **方式一（本机）**：安装 **Node.js 18+**（推荐 LTS），安装时勾选「Add to PATH」。新开终端后执行 `node -v`、`npm -v` 能输出版本即可。
- **方式二（Docker）**：本机只需安装 Docker，无需安装 Node；适合在**两台机器**间用同一套环境。

### 3.2 安装与运行（本机 Node）

```powershell
cd D:\projects\AnotherHistory\AnotherHistory\game
npm install
```

**仅浏览器开发：**

```powershell
npm run dev
```

浏览器打开：http://localhost:5173  

**桌面端（Electron）：**

```powershell
npm run electron:dev
```

会先起 Vite，再打开 Electron 窗口。需本机已装 Node。

### 3.3 安装与运行（Docker，两台机器一致）

在任意一台有 Docker 的机器上：

```powershell
cd D:\projects\AnotherHistory\AnotherHistory\game
docker compose up --build
```

浏览器打开：http://localhost:5173  

代码用 Git 同步到另一台后，在另一台同样执行 `docker compose up` 即可，无需在那台装 Node。  
Electron 桌面版仍需在某一台本机装 Node 后执行 `npm run electron:dev`。

### 3.4 打包与生产运行（Electron）

```powershell
cd game
npm run build
$env:NODE_ENV='production'; npx electron .
```

生产模式会加载 `dist/`，并从 .env / config.json / api_key.txt 读 API Key，供运行时 AI 叙事与异史凝练使用。

### 3.5 内容生成流水线

修改 `design/序章大纲.md` 后生成序章数据：

```powershell
cd game
npm run generate:prologue
```

强制重新生成（忽略缓存）：`npm run generate:prologue:force`  
仅合并不调 AI（已有 nodes + texts 时）：`npm run merge-only`

## 4. 两台机器协作（简要）

- 用 **Git** 同步代码（commit + push / pull）。
- 在两台机器上都可在 **game** 下执行 `npm install` 后跑 `npm run dev` 或 `npm run electron:dev`。
- 若需环境一致，可用 **Docker**：`cd game && docker compose up --build`，浏览器访问即可；Electron 桌面版仍需本机 Node。

## 5. 常见问题速查

| 现象 | 处理 |
|------|------|
| `npm` 无法识别 / 禁止运行脚本 | 安装 Node.js 并加入 PATH；PowerShell 执行策略可设 `Set-ExecutionPolicy Bypass -Scope Process` 或使用 `npm.cmd`。 |
| 游戏里 Failed to fetch | 多为 Electron 下 `file://` 路径或未读到 API Key；确保已 build、且 .env 或 api_key.txt 在根目录或 game 下。 |
| 没有 AI 叙事、只显示骨架文案 | 检查 API Key 是否被读取（根或 game 下 .env、或 game/public/config.json、或 api_key.txt 内含 `sk-...`）。 |
| 流水线报错 No API key | 同游戏：配置 .env（game 或根目录）或 api_key.txt 或 public/config.json。 |
