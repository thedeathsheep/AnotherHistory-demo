# 折戟原大纲

> 与序章 [`序章大纲.md`](序章大纲.md) 并列的界级策划输入；**叙事节点与选项以数据为准**，本文只概括走向与对齐关系。

---

## 单一数据源（避免与 design-seed 重复）

| 内容 | 权威文件 |
|------|----------|
| 节点、`truth_anchors`、`taboo`、`description`、`choices` | [`public/data/skeleton.json`](../public/data/skeleton.json) 内 realm `ZheJiYuan` |
| Engine v2 Planner：`theme`、张力曲线、拍数估计、`forbidden`、**锚点 `anchors`** | 同上文件内该 realm 的 **`planner_seed`** |
| 世界/主角/序章 Planner 条 | [`public/data/design-seed.json`](../public/data/design-seed.json) |

运行时由 `loadDesignSeed(skeleton)` 将 `planner_seed` 合并进 Planner 用的 `DesignSeed`，**勿**再在 `design-seed.json` 里维护第二条 `ZheJiYuan` realm。

---

## 核心元素（与骨架一致）

- **场景**：大雾、铁腥味/肉质腐烂感、地上影子异常、雾中多扇门。
- **禁忌**：不可回头（与节点 `taboo` 一致）。
- **中段**：焚烧坑、草灰涂面、逆风而行；结案标签「焚尸者言」。
- **恶果支**：触犯禁忌后的反噬；残缺结案「焚尸者言（残缺）」。

---

## 大致走向

1. 入口节点：在雾中抉择如何接近真相或退避。
2. 第二节点：见焚烧坑与仪迹，可选正规结案、触碰余烬、或回头（导入恶果）。
3. 恶果节点：付出代价后仍可勉强残缺结案或继续消耗。

---

## 与 Markdown 流水线的关系

- 折戟原 **不**走 `npm run generate:prologue` 那条 AI-1/2/3 章节流水线；内容以 **`skeleton.json` 手改 + 可选 `npm run format:skeleton`** 为主。
- 若日后为折戟原单独接生成管线，合并到 `public/data` 时须保留 **`planner_seed`** 块。

---

## 细纲要求（供扩展节点时自检）

- **summary / plot 导向**：写具体物象与动作，与 `planner_seed.anchors[].must_include` 可对齐检查。
- **选项**：只写角色念头/行动，机制与后果由数据与系统处理。
