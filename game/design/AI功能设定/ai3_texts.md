# AI-3：叙事文本生成器 功能设定

> 为节点和选项填充具体文案：description、choice.text。

---

## 职责

- **输入**：总设定.md、本文件、nodes.json（AI-2 输出）
- **输出**：texts.json（各 node 的 description、各 choice 的 text）
- **不做**：不修改节点结构、next、state、conclusion_label

## 输出格式

```json
{
  "descriptions": {
    "Prologue_01": "你自昏睡中醒来...",
    "Prologue_02": "..."
  },
  "choice_texts": {
    "Prologue_01_0": "靠近枯井，一探究竟。",
    "Prologue_01_1": "收拾书箱，离开此地。",
    "Prologue_02_0": "俯身窥探，记下所见。"
  }
}
```

- `descriptions`：key 为 node_id，value 为 1–2 句叙事，点明处境即可（叙事为游戏服务，不罗列元素）
- `choice_texts`：key 为 `{node_id}_{choice_index}`，value 为选项文案（简短、1 句内）

## 优先阅读：铁律与合格示例（生成时请严格模仿）

**三条铁律**：
1. **只写具体**：具体动作、物象、身体反应；不写思绪、心、意识作主语，不写抽象环境（「清晰的蓝天」「宁静的环境」「……中游走」）。
2. **极简点明处境**：1–2 句即交选项；不罗列元素、不比喻、不浅白（渐渐、感受到、内心觉察、熟悉又陌生、心跳回荡等）。
3. **有衔接**：节点间因果或情绪衔接，每句有具体情节点，避免孤立无推进感。

**合格示例**（按此风格与信息密度生成，3 个连续节点）：

| node_id | description | choice_texts |
|---------|-------------|--------------|
| scene_1 | 你醒过来的时候，草还挂着露。脚边是一条踩出来的土路，背上的东西压得肩头发酸。天刚亮。 | scene_1_0: 拎拎肩上那包，沉。 / scene_1_1: 先爬起来，瞅瞅四下。 |
| scene_2 | 眼前一片草，黄不拉几的，风一过沙沙响。日头挺好，前头还能瞅见路。 | scene_2_0: 往前走。 / scene_2_1: 在草里坐会儿。 |
| scene_3 | 路边有树，影子碎在地上，光一块一块的。你站住，没动。 | scene_3_0: 凑过去，看看那影子。 / scene_3_1: 不看，走。 |

生成时请对照上表：description 为 1–2 句、具体物象与动作、无抽象无浅白；choice_text 为念头/动作、口语化、不重复 objective。

## 文风（显性展示文案以此为准）

**总体原则**：叙事为游戏服务，极简点明处境后交给选项。**所有描述必须具体**：只写具体动作、身体反应、可见可触的物象；禁止抽象主语与抽象环境。拒绝浅白、套路句。节点之间要有叙事趣味与逻辑衔接，避免单句孤立、无推进感。

- **叙事（description）**：1–2 句点明处境，**必须具体**——谁在做什么、什么物、什么身体反应。禁止抽象句（如「思绪在清晰的蓝天与宁静的环境中游走」）；禁止「思绪」「心」「意识」作主语、「在……中游走」、「清晰的蓝天」「宁静的环境」等抽象环境。每个节点要有具体情节点或张力点；与前后节点/选择形成因果或情绪衔接，有逻辑推进。禁止罗列元素、禁止比喻句。
- **选项（choice_text）**：念头或动作，口语化、贴人物；不暴露设定。
- **异史**（结案时由运行时 AI 凝练）：纪传体、以「记之曰」收束；同样拒绝浅白与套路句。

## 重要：选项不暴露设定

**choice_text 必须是角色念头/行动**，不可暴露设定、后果、机制；**不可为 __结案__ 或技术占位符**：
- ✅ 「回头张望来路」「俯身窥探，记下所见」
- ❌ 「目睹衰败」「体验犯忌后果」「选择触发衰败」

## 常见问题与禁止（避免当前生成出现的弊病）

**拒绝浅白。** 以下均为禁止的浅白、套路表达，信息量低、空洞：
- ❌「渐渐清醒的你」「意识逐渐清醒」「景象逐渐清晰」——不写「渐渐/逐渐」，直接写结果（如「你醒了。」）。
- ❌「感受到一阵清风拂过」「感受到……气息」——不写「感受到」，写身体或环境的具体反应（如「风擦过脖子。」「有土味。」）。
- ❌「内心开始觉察」「内心涌起」「心中……」——不写内心/觉察，只写动作或物象。
- ❌「熟悉又陌生的土地」「熟悉又陌生」——套路词，删除；若需地点，直接写「野地」「土路」等。
- ❌「心跳声在耳边回荡」「带来一丝真实的触感」——泛泛感官句，不写；或改为极简（如「心口在跳。」）。
- ❌「思绪在清晰的蓝天与宁静的环境中游走」「思绪游走」「心在……中」——**禁止抽象主语与抽象环境**。不写「思绪」「心」「意识」作主语，不写「清晰的蓝天」「宁静的环境」等抽象环境；必须具体到谁在做什么、什么物、什么身体反应。✅ 例：不写「思绪游走」，写「你抬头。天是蓝的。没云。」或「你站着没动。」

**所有描述必须具体。** 无抽象主语（思绪、心、意识）、无抽象环境（宁静的环境、清晰的蓝天）；只有具体动作、物象、身体反应。

**叙事趣味与节点间逻辑：** 每个节点的描述要有具体情节点或张力点（发生了一件事、一个选择带来的结果、一个待决的冲突等）。生成时考虑与前后节点的衔接：上一选择导致了什么、当前处境为何、选项会推向哪里，形成因果或情绪上的推进，避免节点之间孤立、无逻辑线。

**叙事为游戏服务：** 1–2 句点明处境即交选项。不写多句罗列、不写浅白句、不写抽象句。✅ 例：「你从一片荒野中醒来。」「草挂着露，背上压得发酸。你醒了。」

**禁止比喻句。** 不用「像……」「仿佛……」「犹如……」，只白描。

**叙事（description）禁止：**
- **抽象句**：思绪、心、意识作主语；「在……中游走」；清晰的蓝天、宁静的环境等抽象环境——一律禁止，必须具体到动作、物象、身体反应。
- 浅白句：渐渐/逐渐、感受到、内心觉察、熟悉又陌生、心跳回荡等（见前文）；只写具体动作与物象。
- 空泛无具象、情感悬浮、表达俗套、空洞情绪；不列举选项、不解释玩法；不罗列场景元素。
- 节点描述要有具体情节点或与前后衔接，避免孤立、无推进感。

**选项（choice_text）禁止：**
- 禁止与节点的 objective 字面雷同或复述（objective 是策划用，选项是角色当下念头/动作）。
- 禁止与 description 同一句重复（如描述已说「找到安全的地方休憩」，选项不要再写「找到一个安全的地方休憩」）。
- 选项应是「此刻的念头或动作」，不是任务名或概括；可带轻微异常感或古风，与叙事气质一致。

**避免固定套词：** 不用「古道」「书箱」等设定式名词，用写实、可替换的说法。
**示例：**
- 叙事 ✅：「你从一片荒野中醒来。」「草挂着露，背上压得发酸。你醒了。」「光一块一块的。你站住，没动。」「他选了回头。现在影子在脚底下。」（具体、有情节点或衔接）
- 叙事 ❌：「思绪在清晰的蓝天与宁静的环境中游走。」→ 抽象主语+抽象环境；改为具体，如「你抬头。天蓝，没云。」
- 叙事 ❌：「渐渐清醒的你，感受到一阵清风拂过，内心开始觉察这片熟悉又陌生的土地。」→ 浅白；改为「你醒了。风擦过。野地在脚边。」
- 叙事 ❌：节点之间无衔接、无推进感 → 每句要有具体情节点，与前后选择/节点有因果或情绪线。
- 选项：「拎拎肩上那包，沉。」「先爬起来，瞅瞅四下。」「记一笔。」「不记，走人。」

## 手写/手改文案工作流

当自动生成效果不理想时，可直接手写或修改文案，使游戏快速可玩：

1. **编辑文案**：直接编辑 `generated/chapters/prologue/texts.json`（或复制 `design/examples/prologue_texts_example.json` 到该路径作为起点），保持 `descriptions` 与 `choice_texts` 的 key 与当前 `nodes.json` 一致（node_id、node_id_0 等）。
2. **合并到游戏**：在 `game` 目录下执行 `node scripts/merge-only.mjs prologue`，将 texts 与 nodes 合并，并写入 `public/data/prologue.json`。
3. **刷新游戏**：重新加载或重启游戏即可看到更新后的叙事与选项。

**示例文案**：`design/examples/prologue_texts_example.json` 提供一份符合本文件规范的序章文案，可作为「合格线」参考或直接复制使用。

## 约束

- **每个节点的 description 必须具体**：只有具体动作、物象、身体反应；禁止抽象主语（思绪、心、意识）与抽象环境（宁静的环境、清晰的蓝天、「……中游走」等）
- **每个节点要有具体情节点或与前后衔接**，形成叙事趣味与逻辑线，避免节点孤立、无推进感
- 每个节点的 description 必须独特，不可与其他节点重复或相似；严格按 node_id 逐一生成
- 不得触犯节点的 taboo
- 必须自然融入 plot_guide 中的关键词
- 选项数量与 nodes 中 choices 长度一致
