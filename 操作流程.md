## 2. API Key 配置（运行时 AI + 离线流水线）

**游戏运行时**（任选其一）：

- `game/.env`：`VITE_AIHUBMIX_API_KEY=sk-…`
- `game/public/config.json`（**不要提交**）：`{ "aihubmixApiKey": "sk-…" }`
- 环境变量：`AIHUBMIX_API_KEY` / `VITE_AIHUBMIX_API_KEY`

**离线生成** `prologue`：`npm run generate:prologue` 使用 `scripts/utils.mjs` 的 `getApiKey()`，与上相同。

## 3. 游戏（`game/`）安装与运行

### 3.1 本机 Node

- 安装 **Node.js 18+**（LTS），勾选加入 PATH。

```powershell
cd <仓库根目录>\game
npm install
npm run dev
```

浏览器：<http://localhost:5173>

**Electron：**

```powershell
npm run electron:dev
```

### 3.2 Docker

```powershell
cd <仓库根目录>\game
docker compose up --build
```

浏览器同样打开 <http://localhost:5173>。Electron 仍需本机 Node 执行 `npm run electron:dev`。

### 3.3 打包

```powershell
cd game
npm run build
```

产物在 `game/dist/`。

## 5. 常见问题速查

| 现象 | 处理 |
|------|------|
| `npm` 无法识别 | 安装 Node.js 并加入 PATH；新开终端。 |
| 有 Key 但境遇正文一直不出 / 只显示骨架 | 查网络、Key 是否有效、控制台报错；失败时会回退骨架 `description`。叙事加载中文案为 **「境遇正文凝练中…」**（不是「正在感应」）。 |
| 感应区显示「待正文落定，感应方显。」 | 正常：当前节点走 AI 叙事时，**正文入缓存后**才列出【感应】与选项。 |
| import 报错（如 `requests`） | 游戏是 Node 项目，勿在 `game/` 里用 Python 依赖；在 `game` 下执行 `npm install`。 |
