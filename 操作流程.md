## 2. API Key 配置（AI 生成用）

- **Key 文件**：在项目根目录维护 `api_key.txt`，把 AIHubMix 的 key 写在里面（例如文档里有一行 `api_key="sk-..."`，解析时会用）。
- **原型（prototype）**：自动从上级目录的 `api_key.txt` 读取，无需再配。
- **游戏（game）**：二选一即可：
  - 在 `game/` 下新建 `.env`，内容：`VITE_AIHUBMIX_API_KEY=sk-你的key`
  - 或在 `game/public/` 下新建 `config.json`（不要提交），内容：`{ "aihubmixApiKey": "sk-..." }`

## 3. 原型（Python + Textual）完整流程

### 3.1 环境

- 已安装 **Python 3.10+** 并加入系统 PATH。
- 若未安装：Windows 可用 `winget install Python.Python.3.12`，安装后**新开终端**或重启 Cursor。

### 3.2 安装依赖

在项目根下执行（若遇 SOCKS/代理报错，用下面带 `NO_PROXY` 的一行）：

```powershell
cd D:\projects\AnotherHistory\AnotherHistory\prototype
pip install -r requirements.txt
```

若报错 `Missing dependencies for SOCKS support`，改为：

```powershell
$env:NO_PROXY='*'; $env:no_proxy='*'
pip install -r requirements.txt --trusted-host pypi.org --trusted-host files.pythonhosted.org
```

（或先执行前两行设置环境变量，再执行第三行。）

### 3.3 运行

```powershell
cd D:\projects\AnotherHistory\AnotherHistory\prototype
python app.py
```

- **上下键**：在感应选项中移动  
- **Enter**：确认当前选项  
- **q**：退出  

无 API Key 时用骨架文案；有 Key 时节点叙事与结案异史会走 AI，界面会标「【AI 生成】」「【AI 凝练】」。

## 4. 游戏（方案 A）完整流程

### 4.1 环境

- **方式一（本机）**：安装 **Node.js 18+**（推荐 LTS），安装时勾选「Add to PATH」。新开终端后执行 `node -v`、`npm -v` 能输出版本即可。
- **方式二（Docker）**：本机只需安装 Docker，无需安装 Node；适合在**两台机器**间用同一套环境。

### 4.2 安装与运行（本机 Node）

```powershell
cd D:\projects\AnotherHistory\AnotherHistory\game
npm install
```

**仅浏览器开发：**

```powershell
npm run dev
```

浏览器打开：http://localhost:5173  

**桌面端（Electron）：**

```powershell
npm run electron:dev
```

会先起 Vite，再打开 Electron 窗口。需本机已装 Node。

### 4.3 安装与运行（Docker，两台机器一致）

在任意一台有 Docker 的机器上：

```powershell
cd D:\projects\AnotherHistory\AnotherHistory\game
docker compose up --build
```

浏览器打开：http://localhost:5173  

代码用 Git 同步到另一台后，在另一台同样执行 `docker compose up` 即可，无需在那台装 Node。  
Electron 桌面版仍需在某一台本机装 Node 后执行 `npm run electron:dev`。

### 4.4 打包

```powershell
cd game
npm run build
```

产物在 `game/dist/`，可交给静态服务器部署。

## 5. 两台机器协作（简要）

- 用 **Git** 同步代码（commit + push / pull）。
- 游戏前端：在两台机器上都用 **Docker**（`docker compose up`）可保证 Node/依赖一致，无需各自装 Node。
- 原型：两台机器各装 Python，拉代码后执行 `pip install -r requirements.txt` 和 `python app.py`；若遇 pip 代理问题，用第 3.2 节的 `NO_PROXY` 方式安装。

## 6. 常见问题速查

| 现象 | 处理 |
|------|------|
| 终端里 `pip` / `python` 找不到 | 安装 Python 并勾选加入 PATH；或新开终端、重启 Cursor。 |
| `npm` 无法识别 | 安装 Node.js 并勾选加入 PATH；新开终端再试。 |
| pip 报错 `Missing dependencies for SOCKS support` | 使用第 3.2 节的 `NO_PROXY` + `--trusted-host` 再执行 pip install。 |
| 原型一直显示「正在感应」、没有 AIGC | 多为 API 超时或失败；已做失败时回退骨架 + 重试。若仍无 AIGC，检查 api_key.txt 和网络。 |
| 游戏里 import 报错（如 `requests`） | 原型是 Python，需在 prototype 下执行 `pip install -r requirements.txt`；游戏是 Node，用 `npm install`。 |
| Textual 报错 `the first argument must be callable` | 已修复：`call_later` 第一个参数是回调函数，不要传数字（如 0）。 |
